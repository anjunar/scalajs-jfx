Scala.js UI architecture
Typed browser apps that keep their shape as they grow.
scalajs-jfx brings a JavaFX-inspired component model to Scala.js: routed pages, typed form state, remote data views, secondary windows and live reference material all follow one composable DSL.

Architecture
A small set of primitives carries the whole application.
The project is easiest to understand as a chain: routing creates the workspace, components own precise state, data surfaces react to changes, and the viewport keeps secondary work in context.

Value
The benefits are practical: less glue, clearer boundaries, safer UI state.
scalajs-jfx is useful when a browser app needs real structure without giving up the speed and reach of Scala.js.

Project purpose
A precise Scala UI toolkit for applications that need structure before they need spectacle.
The repository is both framework and proof: a compact runtime, a coherent demo app and a living reference that show how Scala.js can carry serious browser interfaces with architectural calm.
