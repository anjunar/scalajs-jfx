Forms
jfx.form.Form
Form
Typed form composition around a model instance. Forms bind controls to model properties and keep validation, submission and nested composition in one place.
Operational Fit
Works well with scoped model injection.
Nested subforms can bind to object properties without extra ceremony.
A good fit for admin forms, editors and detail pages.
API Snapshot
form(model) { ... }
Create a typed form for a model instance.
onSubmit
Handle form submission inside the DSL.
valueProperty
Observe the active model bound to the form.
SubForm
Bind nested objects into the same editing flow.
Import Surface
import jfx.dsl.Scope.{inject, scoped} import jfx.form.Form.form import jfx.json.JsonMapper
Usage Skeleton
scope { singleton[JsonMapper] { new JsonMapper() } scoped[Person] { inject[JsonMapper].deserialize[Person](JSON.parse(json)) } form(inject[Person]) { onSubmit = _ => println(JSON.stringify(inject[JsonMapper].serialize(inject[Person]))) } }
Integration Patterns
Scoped Model Setup

Nested Editing
Compose child objects as subforms so submission still produces one coherent model.
Incremental Richness
