Action
jfx.control.Link
Link
Client-side navigation links with history support. Links provide a way to navigate between pages without full browser reloads, integrating with the browser history.
Operational Fit
Prevents default browser navigation for client-side routing.
Supports custom click handlers for additional logic.
Consistent with standard HTML anchor tags for accessibility.
API Snapshot
link(href) { ... }
Create a navigation link.
href
Set the target URL for the link.
onClick { event => ... }
Add custom logic to the link click.
Import Surface
import jfx.control.Link.link
Usage Skeleton
link("/docs/router") { textContent = "Go to Router Docs" }
Integration Patterns
Internal Navigation
Use links for moving between application routes handled by the Router.
Breadcrumbs
Compose links in a horizontal row for clear navigation paths.
Action Links
Use links for lightweight actions that feel more like navigation than primary triggers.
