Action
2 references
jfx.action.Button
Button
A standard clickable button for triggering actions and form submissions.
Buttons are the primary way for users to interact with your application logic or submit forms.
Form Submission: Buttons inside a form default to 'submit' type, triggering the form's onSubmit handler.
jfx.control.Link
Link
Client-side navigation links with history support.
Links provide a way to navigate between pages without full browser reloads, integrating with the browser history.
Internal Navigation: Use links for moving between application routes handled by the Router.

jfx.router.Router
Router
Declarative route handling for page-sized DSL components.
Use the router when the app needs a small, composable navigation model without leaving the Scala DSL.
App Shell: Register the router once in application scope and mount it inside the viewport.

jfx.control.Heading
Heading
Semantic text headings from H1 to H6.
Headings help structure your content and provide visual hierarchy to your pages.
Page Titles: Use Heading(1) for the main title of a page.
jfx.control.Image
Image
Display images with reactive source and styling.
Image provides a way to render images while observing source changes and providing easy styling hooks.
User Avatars: Bind an image to a user's profile picture URL.
Data
3 references
jfx.core.state.ListProperty.remote
RemoteListProperty
A state primitive for lazy, remote-looking list loading.
Use remote list properties when you want a collection API that can page, sort and refetch from a loader.
Backend Simulation: Model real loading behaviour early so the UI is already ready for an API later.
jfx.control.TableView
TableView
Structured data presentation with sorting, virtualization and custom cells.
The table API is designed for real application data, not just static samples.
Column Ownership: Keep sorting and display decisions near the column so the model mapping stays easy to read.
jfx.control.VirtualListView
VirtualListView
High-performance list rendering for large datasets using virtualization.
VirtualListView only renders the items currently visible in the viewport, allowing it to handle thousands of rows with ease.
Large Data Sets: Use virtualization when you have more than 50-100 items to keep the DOM lean.
Forms
5 references
jfx.form.ComboBox
ComboBox
A customizable selection control with renderable values and dropdown rows.
The combo box API supports custom item rendering, custom selected-value rendering and overlay-based dropdown behaviour.
Custom Brand Feel: Render both the selected value and the dropdown rows so the control matches the rest of the product.
jfx.form.Editor
Editor
Rich content editing inside the same form runtime.
The editor keeps structured writing, toolbar state and plugin modules inside one composable control.
Content Authoring: Use it when the user needs a structured writing surface rather than a plain textarea.
jfx.form.Form
Form
Typed form composition around a model instance.
Forms bind controls to model properties and keep validation, submission and nested composition in one place.
Scoped Model Setup: Create the working model once per page or dialog and inject it where controls need it.
jfx.form.ImageCropper
ImageCropper
Media editing inside a typed form workflow.
The image cropper is meant for workflows where choosing and shaping media should stay inside the same model-editing surface.
Profile And CMS Flows: Use it when upload, crop and preview should all stay inside one form experience.
jfx.form.InputContainer
InputContainer
A layout wrapper that gives controls labels, states and error presentation.
Input containers keep form rows visually consistent while reacting to control state such as focus, invalid state and emptiness.
Consistent Rows: Wrap every control in the same container to keep spacing, labels and validation behaviour aligned.
Layout
5 references
jfx.layout.{HBox, VBox}
Box (HBox / VBox)
Fundamental horizontal and vertical flex-based containers.
HBox and VBox provide a simple way to align children in a row or column using CSS flexbox.
Toolbars: Use HBox to align buttons and controls in a single row.
jfx.layout.{Div, Span}
Div / Span
Basic HTML block and inline containers.
Div and Span are the fundamental building blocks for custom layouts and text styling.
Custom Containers: Use div to create unique layout structures not covered by HBox/VBox.
jfx.layout.Drawer
Drawer
A sliding side panel for navigation and supplemental content.
Drawers slide in from the edge of the screen, providing space for navigation menus or detail views without losing context.
Mobile Navigation: Hide the main menu in a drawer on smaller screens.
jfx.layout.Viewport
Viewport
A shared layer for windows, notifications and overlays.
Viewport lets the app keep richer interaction layers inside the same runtime and visual shell.
App-Level Host: Mount the viewport once near the shell so every page can open notifications and windows.
jfx.layout.Window
Window
Floating, draggable windows for secondary workflows.
Windows provide a desktop-like experience within the browser, allowing for multi-tasking and non-blocking workflows.
Multi-tasking: Allow users to keep an inspector or chat window open while they work.
Logic
4 references
jfx.statement.Conditional
Conditional (when)
Reactive branching in the DSL based on a boolean property.
Use when you need to show or hide parts of the UI based on state changes.
Feature Toggles: Use conditional to hide features that are not yet enabled for the user.
jfx.statement.DynamicOutlet
DynamicOutlet (mount)
A mount point for dynamic component instances.
DynamicOutlet allows you to swap out entire component instances at a specific point in the layout.
Page Navigation: The foundation of client-side routing and page transitions.
jfx.statement.ForEach
ForEach
Efficiently render a list of items from a ListProperty.
ForEach observes a ListProperty and incrementally updates the DOM as the list changes (add, remove, move).
Dynamic Lists: Perfect for todo lists, chat messages or any collection that changes over time.
jfx.statement.ObserveRender
ObserveRender
Reactive component rebuilding based on a property value.
ObserveRender re-executes its inner DSL block whenever the source property changes, replacing its entire content.
Detail Views: Rebuild the entire detail pane when the selected item changes.

Mounting

Injected Components: Mount routers, controllers or shared widgets that were created outside the local DSL block.
jfx.dsl.Scope
Scope
Hierarchical dependency injection and lifecycle management.
Scopes allow you to register and inject services or model instances throughout a component tree.
Service Locator: Provide global services like API clients or event buses at the root scope.
Forms, tables, windows and docs all run on the same component model.
